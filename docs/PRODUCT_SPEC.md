# מסמך אפיון מערכת (PRD) - SPARK Quality מבית SPARK AI

**תאריך עדכון:** מאי 2026
**גרסה:** 1.0 (MVP)
**מחבר:** צוות SPARK AI (יפעת איתן וענת גרינברג)

---

## 1. מטרת המערכת
SPARK Quality היא מערכת SaaS (Software as a Service) מבוססת בינה מלאכותית המיועדת לסוכני ביטוח וסוכנויות ביטוח בישראל. המערכת נועדה להחליף את העבודה הידנית והרפטטיבית של ניתוח דוחות אקסל (כגון דוחות שורנס / מסלקה), זיהוי הזדמנויות עסקיות, וניהול קשרי לקוחות (CRM) באמצעות אוטומציות חכמות וניסוח הודעות מותאמות אישית על ידי AI.

## 2. ארכיטקטורה וטכנולוגיה
המערכת בנויה בארכיטקטורת Multi-Tenant (ריבוי דיירים), כך שכל סוכנות מקבלת סביבת עבודה (Workspace) מבודדת ומאובטחת.

- **Frontend:** React 19, Tailwind CSS 4, shadcn/ui
- **Backend:** Node.js (Express), tRPC 11
- **Database:** MySQL / TiDB (ניהול סכמות באמצעות Drizzle ORM)
- **Authentication:** Manus OAuth (התחברות מאובטחת)
- **AI Integration:** חיבור מובנה למודלי שפה (LLM) לניסוח הודעות וניתוח נתונים.

### 2.1 מודל נתונים (Data Model)
- **Workspaces:** סביבת העבודה של הסוכנות (כולל הגדרות כמו סף VIP).
- **Users:** סוכנים ומנהלים. הרשאות: `owner` (בעלים), `admin` (מנהל סוכנות - רואה הכל), `agent` (סוכן - רואה רק את לקוחותיו).
- **Clients:** לקוחות הקצה (מזוהים לפי ת.ז, ייחודיים ברמת ה-Workspace).
- **Policies:** פוליסות הביטוח והפיננסים של הלקוחות.
- **Reports:** היסטוריית דוחות שהועלו למערכת.
- **Action Items:** משימות אוטומטיות שנוצרו על ידי המערכת (למשל: "פוליסת ריסק מסתיימת").

## 3. זרימת משתמש (User Flow)

### 3.1 דף נחיתה (Landing Page)
- **כתובת:** `/`
- **מטרה:** שיווק המערכת לסוכנויות חדשות.
- **תכולה:** Hero סינמטי, סטטיסטיקות (זמן ניתוח, פוטנציאל הכנסה), הסבר על יכולות ה-AI, וכפתורי הנעה לפעולה (התחברות / צפייה בדמו).

### 3.2 תהליך קליטה (Onboarding)
- **כתובת:** `/onboarding`
- **מטרה:** הגדרת סביבת העבודה לסוכן חדש.
- **שלבים:**
  1. **יצירת סביבת עבודה:** הזנת שם הסוכנות.
  2. **הזמנת צוות:** אפשרות לשלוח הזמנות לסוכנים נוספים (מייצר קישור הזמנה ייחודי).
  3. **הגדרות ראשוניות:** קביעת סף VIP (ברירת מחדל: 1,000,000 ₪).

### 3.3 העלאת דוחות (Upload Report)
- **כתובת:** `/upload`
- **מטרה:** הזנת נתונים למערכת.
- **תהליך:** הסוכן מעלה קובץ אקסל (למשל, דוח שורנס). המערכת מנתחת את הקובץ בזמן אמת, מזהה לקוחות, פוליסות, ודגלים אדומים/הזדמנויות.

### 3.4 דשבורד מרכזי (Dashboard)
- **כתובת:** `/dashboard`
- **מטרה:** תמונת מצב יומית של הסוכנות.
- **תכולה:**
  - מדדים מרכזיים (סה"כ לקוחות, לקוחות VIP, הון נזיל, פוטנציאל תיקון 190, סה"כ נכסים מנוהלים - AUM).
  - חלוקה לקטגוריות (ראה סעיף 4).
  - טבלת לקוחות עם אפשרויות סינון וחיפוש.

### 3.5 ניהול לקוחות (Clients)
- **כתובת:** `/clients`
- **מטרה:** צפייה בתיק הלקוחות המלא.
- **תכולה:** רשימת לקוחות, סטטוסים, ופעולות מהירות (שליחת הודעה, יצירת משימה).

### 3.6 ניהול צוות (Team)
- **כתובת:** `/team`
- **מטרה:** ניהול הרשאות והזמנת סוכנים (למנהלים בלבד).

### 3.7 תמחור (Pricing)
- **כתובת:** `/pricing`
- **מטרה:** הצגת מסלולי המנוי (Base Plan, Premium Plan).

## 4. קטגוריות ותרחישים עסקיים (Scenarios)
המערכת מזהה אוטומטית 6 קטגוריות מרכזיות מתוך הנתונים, ומציעה פעולות המשך:

1. **לקוחות VIP (ניהול עושר)**
   - **זיהוי:** לקוחות עם צבירה כוללת מעל סף ה-VIP (למשל 1M ₪).
   - **פעולה:** תיאום פגישת תכנון פיננסי מקיפה, הצעת מוצרי פרימיום.
2. **תשואות נמוכות (שימור והוזלת דמי ניהול)**
   - **זיהוי:** פוליסות עם תשואה מתחת לממוצע (היוריסטיקה נוכחית: מזהה מילות מפתח בדוח).
   - **פעולה:** פנייה יזומה ללקוח להצעת מסלול חלופי או הוזלת דמי ניהול למניעת נטישה.
3. **תיקון 190 (פטור ממס רווחי הון)**
   - **זיהוי:** לקוחות מעל גיל 60 עם הון פנוי.
   - **פעולה:** שליחת סימולציה המדגימה חיסכון במס רווחי הון בעת משיכה.
4. **ריסק זמני (חידוש כיסוי בזמן)**
   - **זיהוי:** פוליסות ריסק (חיים/בריאות) שמסתיימות ב-30-60 הימים הקרובים.
   - **פעולה:** התראה דחופה לסוכן ושליחת הודעת חידוש ללקוח.
5. **תום הנחה (שימור פרמיה)**
   - **זיהוי:** פוליסות שהנחת דמי הניהול/פרמיה שלהן עומדת להסתיים.
   - **פעולה:** פנייה ללקוח לבחינת חידוש ההטבה או מעבר למוצר עדיף.
6. **חוסרים בכיסויים (השלמת תיק)**
   - **זיהוי:** לקוחות ללא כיסויים בסיסיים (למשל, ללא פנסיה, ללא ריסק לחיים).
   - **פעולה:** הצעת מוצרים משלימים בהתאמה אישית (Cross-sell).

## 5. מודול AI Composer (ניסוח הודעות)
המערכת כוללת מודאל חכם המנסח הודעות אימייל ו-WhatsApp מותאמות אישית לכל לקוח, בהתבסס על הסטטוס שלו.
- **תהליך החשיבה של ה-AI:** קורא נתוני לקוח -> מנתח היסטוריה -> מזהה טון מועדף -> מנסח הודעה.
- **תוצר:** טקסט מוכן לשליחה, הכולל את שם הלקוח, פרטי הפוליסה, וקריאה לפעולה (Call to Action) ברורה.

## 6. סטטוס פיתוח ופערים (Gap Analysis)

### 6.1 פותח והושלם (Done)
- ארכיטקטורת Multi-Tenant מלאה.
- מערכת הרשאות (Owner, Admin, Agent).
- דפי נחיתה, Onboarding, דשבורד, ניהול לקוחות וצוות.
- מנגנון העלאת דוחות (Mock/Parsing ראשוני).
- מודל נתונים מלא ב-Drizzle.
- חוויית דמו סינמטית (`/demo?clean=true`).
- מודול AI Composer (תבניות מבוססות סטטוס).

### 6.2 חסר / דורש השלמה (Pending / Blocked)
1. **חיבור API למסלקה הפנסיונית / שורנס:**
   - **סטטוס:** חסום.
   - **סיבה:** דורש חתימה על חוזה מסחרי מול הגופים וקבלת מפתחות API.
   - **פתרון זמני:** העלאת קבצי אקסל ידנית (Upload Report).
2. **בנצ'מרק תשואות מדויק:**
   - **סטטוס:** חסום.
   - **סיבה:** נגזרת של סעיף 1. דורש גישה למאגר נתונים היסטורי של תשואות.
   - **פתרון זמני:** זיהוי היוריסטי (מבוסס טקסט/חוקים פשוטים) בעת העלאת הדוח.
3. **אינטגרציית תשלומים (Billing):**
   - **סטטוס:** ממתין.
   - **סיבה:** העדפה למערכת iCount על פני Stripe.
   - **פעולה נדרשת:** אפיון ופיתוח חיבור ל-API של iCount לניהול מנויים והפקת חשבוניות.
4. **אוטומציות חיצוניות (Make / Zapier):**
   - **סטטוס:** תכנון עתידי.
   - **פעולה נדרשת:** חשיפת Webhooks מהמערכת כדי לאפשר לסוכנים לחבר את SPARK Quality למערכות ה-CRM הקיימות שלהם (למשל, יצירת משימה ב-Salesforce כשמזוהה לקוח VIP).

## 7. סיכום
SPARK Quality מציגה MVP חזק ומוכן להדגמה. המערכת מדגימה בצורה ברורה את הערך המוסף של AI בייעול עבודת סוכן הביטוח. השלבים הבאים יתמקדו ביצירת שיתופי פעולה טכנולוגיים (APIs) להזנת נתונים אוטומטית וחיבור למערכת סליקה מקומית.
